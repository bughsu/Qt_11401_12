# 🌐 遠端 Web 監看快速指南

## 一分鐘快速開始

### 步驟 1: 啟動應用程式並新增串流
1. 執行 Qt 應用程式
2. 點擊「新增來源」
3. 輸入串流 URL 或影片路徑
4. 點擊「開始播放」

### 步驟 2: 啟用遠端監看
1. 點擊「**啟用遠端監看**」按鈕
2. 等待系統顯示連接網址

```
┌─────────────────────────────┐
│  啟用遠端監看 ✓             │
├─────────────────────────────┤
│ 網址: http://192.168.1.100:8080
│                              │
│ 用手機掃描此 QR Code 或     │
│ 直接輸入網址瀏覽            │
└─────────────────────────────┘
```

### 步驟 3: 手機連接
1. 確保手機與電腦在**同一個 Wi-Fi** 網路
2. 開啟手機瀏覽器（Safari / Chrome）
3. 輸入顯示的網址，例如：`http://192.168.1.100:8080`
4. 即刻觀看即時監控畫面！

## 🔧 常見問題

### Q: 手機無法連接？
**A:** 檢查以下項目：
1. 手機和電腦是否在同一個區域網路（Wi-Fi）
2. 防火牆是否封鎖了 Port 8080
3. 電腦 IP 地址是否正確（可能因為 DHCP 變動）

### Q: 影像延遲或卡頓？
**A:** 可能的原因：
1. Wi-Fi 信號弱 → 靠近路由器
2. 網路頻寬不足 → 關閉其他下載/串流
3. CPU 負載過高 → 減少同時播放的串流數量

### Q: 如何同時讓多人觀看？
**A:** 
- 多個裝置可以同時連接同一個網址
- 每增加一個觀看者會稍微增加 CPU 和網路使用
- 建議最多 5-10 個同時觀看者

### Q: 安全性如何？
**A:**
- 目前使用 HTTP（未加密）
- **僅建議在可信任的區域網路使用**
- 未來版本將支援 HTTPS 和密碼保護

## 💡 使用技巧

### 最佳網路設定
```
電腦 ←─ 有線網路 ─→ 路由器
                      ↓
手機 ←─ Wi-Fi 5GHz ─→ 
```

### 提升效能
- 使用 5GHz Wi-Fi（較少干擾）
- 電腦使用有線網路連接
- 關閉其他網路應用程式

### 多畫面監看
- 當有多個串流時，點擊畫面可聚焦
- Web 端會自動顯示聚焦的畫面
- 未聚焦時顯示第一個串流

## 🎯 進階功能

### 自訂 Port
預設 Port 為 8080，如需修改：
```cpp
// 在 mainwindow.cpp 的 onToggleWebServer() 中
m_webServer->start(8080);  // 改為其他 Port，例如 9000
```

### 調整影像品質
```cpp
// 在 webserver.cpp 的 imageToJpeg() 中
image.save(&buffer, "JPEG", 85);  // 85 為品質 (0-100)
```

### 調整幀率
```cpp
// 在 webserver.cpp 的 WebServer 建構函式中
m_frameTimer->setInterval(33);  // 33ms ≈ 30 FPS
                                // 50ms ≈ 20 FPS
                                // 100ms ≈ 10 FPS
```

## 📊 效能指標

### 典型資源使用（1080p 串流）
- **CPU**: +5-10%
- **網路**: 2-5 Mbps per client
- **記憶體**: +10-20 MB

### 建議配置
- **最低**: i3 處理器, 4GB RAM, 100Mbps 網路
- **建議**: i5 處理器, 8GB RAM, 1Gbps 網路
- **最佳**: i7 處理器, 16GB RAM, 有線網路
